So what the hell is an observable? An observable is a simplistic name for an asynchronous data stream. And what exactly does this mean? Well, Sherlock, let's break it down.

  • Asynchronous: Asynchronous functions operate outside of the main thread, meaning we're no longer linear like most code. Instead, we have to wait for the results to become available.
  • Data: Data is data. If you can build it in Angular as an object, it can be passed as data. This includes numbers, arrays, complex objects, functions, anything.
  • Stream: Stream is a fancy way of saying "I have all my data, just not right now". The data becomes available as it becomes available, which is like saying "When I know, you'll know."

So with these definitions, we can come up with the definition of an observable: an observable is a way of handling data as it becomes available. In layman's terms, we can observe it. Well, definitions are all well and good, but let's see an observable in action. In this example, we will loop through an array of colors at different random time intervals (maximum 5 seconds) and pass the hex code as the data aspect of the observable. We will consume the data from the observable through the subscribe function, which essentially means "listen for changes". An important thing to note about observables is that they are not the value you are trying to read. An observable facilitates retrieving this value.
